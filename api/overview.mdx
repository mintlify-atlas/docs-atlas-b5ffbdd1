The CodeAura API is a JSON-over-HTTP API that lets you create shareable code snapshots, manage session history, and authenticate users — the same operations the web app uses under the hood. You can call it directly from your own tooling, CI pipelines, or editor extensions.

## Base URL

All API requests are made to your CodeAura server. For self-hosted deployments, replace the placeholder with your own domain:

```
https://your-domain/
```

The hosted version of CodeAura is available at `https://codeaura.fun`.

## Response format

Every endpoint returns `application/json`. Successful responses include the relevant resource fields. Error responses always carry an `error` string:

```json
{
  "error": "Missing or invalid Authorization header"
}
```

## Rate limiting

The API enforces a limit of **100 requests per 15-minute window**, keyed by client IP address (using the `X-Forwarded-For` or `X-Real-IP` header). The response includes standard `RateLimit-*` headers (`draft-6` format) so you can track your remaining quota.

When you exceed the limit, the server responds with `429 Too Many Requests`.

## Authentication

Authentication is **optional** for snapshot creation — anonymous users can share code without signing in. It is **required** for all history and user-session endpoints.

Tokens are obtained through GitHub OAuth and passed as a Bearer token in the `Authorization` header. See the authentication page for full details.
